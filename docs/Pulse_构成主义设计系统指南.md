# Pulse 看板 — 构成主义设计系统指南

> 版本: v1.0 | 最后更新: 2026-06-19
> 本指南将 1920-1930 年代俄罗斯构成主义视觉语言融入实时监控看板，形成一套可直接作为前端开发约束的设计规范。

---

## 1. 核心风格定位

**一句话概括：**

> 一个「红黑白的工业监控看板」—— 用构成主义的极端对比、硬边几何、对角线张力，呈现实时系统数据的力与秩序。

**关键词**：红黑对撞、全大写粗体、90 度硬边、45 度斜切、机械感动画、增量实时渲染。

**体验目标**：
- 5 秒内看懂全局系统健康状态
- 每个视觉元素都有功能目的，不装饰
- 数据信息的传递像海报一样有力、直接、不容置疑

## 2. 色彩系统

### 2.1 主色板（核心三色）

| 令牌名 | 角色 | HEX | RGB | 在看板中的用途 |
|--------|------|-----|-----|---------------|
| `--color-red` | 革命红 / 主色 | `#CC0000` | 204,0,0 | 关键告警、标题栏背景、折线数据线、楔形装饰、选中态 |
| `--color-black` | 绝对黑 | `#000000` | 0,0,0 | 页面背景（暗色模式）、卡片边框、正文、坐标轴 |
| `--color-white` | 纯白 | `#FFFFFF` | 255,255,255 | 卡片背景、反白文字、负空间 |

### 2.2 扩展灰色阶

| 令牌名 | 角色 | HEX | 在看板中的用途 |
|--------|------|-----|---------------|
| `--color-grey-10` | 极深灰 | `#1A1A1A` | 暗色模式次要背景（与纯黑微区分） |
| `--color-grey-30` | 深灰 | `#4D4D4D` | 次要文字、禁用标签 |
| `--color-grey-50` | 中灰 | `#808080` | 分割线、占位符文字 |
| `--color-grey-70` | 浅灰 | `#B3B3B3` | 次要装饰、非活跃元素 |
| `--color-grey-90` | 极浅灰 | `#E6E6E6` | 图表网格线、卡片悬停态背景 |

### 2.3 功能色（低频使用）

| 令牌名 | 角色 | HEX | 在看板中的用途 |
|--------|------|-----|---------------|
| `--color-yellow` | 警示黄 | `#FFD700` | 警告状态（非严重）、星装饰 |
| `--color-green` | 在线绿 | `#2D8A2D` | 已连接/在线状态指示器 |
| `--color-blue` | 工业蓝 | `#0044CC` | 二级数据线（极少使用） |

### 2.4 色彩使用铁律

1. **红黑占比 90% 以上**，黄色/绿色仅用于状态指示（面积 < 5%）
2. **红色永远不作大面积背景**，仅用于标题栏横条、几何装饰、数据线、告警态
3. **黑底白字 或 白底黑字** 二选一作为基调，不混合
4. **数据中的红色 = 关键/告警/异常**，绿色 = 正常/在线，灰色 = 离线/禁用
5. **图表最多 3 色**（红、黑、灰），必要时加黄色以示警告

## 3. 字体系统

### 3.1 字体栈

| 层级 | 字体栈（fallback 链） | 来源 | 在看板中的用途 |
|------|----------------------|------|---------------|
| `--font-display` | `'Rodchenko', 'Tektur', 'Bebas Neue', 'Impact', sans-serif` | Rodchenko(商业)/Tektur(GFonts OFL) | 主标题、看板名、关键数据标签（全部大写） |
| `--font-heading` | `'Tektur', 'Bebas Neue', 'Montserrat', sans-serif` | Google Fonts | 卡片标题、导航项、Tab 标签（全部大写） |
| `--font-body` | `'Montserrat', 'Inter', sans-serif` | Google Fonts | 正文说明、元数据、辅助文字（全大写） |
| `--font-mono` | `'JetBrains Mono', 'Fira Code', monospace` | Google Fonts OFL | 所有数值、时间戳、日志、数据表内容 |

### 3.2 字号层级

| 令牌名 | 大小 | 在看板中的用途 |
|--------|------|---------------|
| `--text-xs` | 0.75rem (12px) | 元数据、数据新鲜度标签、图表数据点标注 |
| `--text-sm` | 0.875rem (14px) | 事件流条目、图表轴标签 |
| `--text-base` | 1rem (16px) | 卡片辅助文字、导航文字 |
| `--text-lg` | 1.25rem (20px) | 卡片标题、Tab 标签 |
| `--text-xl` | 1.5rem (24px) | 小标题、图表标题 |
| `--text-2xl` | 2rem (32px) | 大标题、指标分组标题 |
| `--text-3xl` | 2.5rem (40px) | 看板主标题 |
| `--text-4xl` | 3.5rem (56px) | 关键 KPI 数值（最重要的 1-2 个指标） |
| `--text-5xl` | 5rem (80px) | 全屏模式下的主 KPI（极少使用） |

### 3.3 排版规则

1. **标题全部大写（ALL CAPS）** — Tektur/Rodchenko 本身无小写，所有卡片标题、导航、Tab、标签都全大写
2. **字号极端对比** — 关键 KPI 数值（`--text-4xl` 56px）与元数据标签（`--text-xs` 12px）之间差距 4.6 倍
3. **字母间距加宽** — 标题 `letter-spacing: 0.15em` 到 `0.2em`，数值 `letter-spacing: 0.05em`
4. **行距紧凑** — `--leading-tight: 0.9`（标题）、`--leading-normal: 1.1`（数值）
5. **数值用等宽字体** — 所有数字、指标、百分比、时间戳必须使用 `--font-mono`，确保数字对齐稳定
6. **文字可被几何切割** — 允许用 clip-path 或叠加几何体截断文字，创造构成主义特有的"图形化文字"

### 4.3 卡片设计

#### 4.3.1 标准指标卡片

```
┌──────────────────────────────────────┐
│  ▐ STATUS: ACTIVE                    │ ← --font-heading, --text-lg, ALL CAPS
│  ┌──────────────────────────────┐    │   左侧 4px 红色竖线装饰
│  │                     +8.2% ▲ │    │ ← 主指标: --font-mono, --text-4xl
│  │              98243        │    │    趋势箭头: 红/绿/灰
│  │                    ↓ -2.1% │    │    (正三角=上升, 倒三角=下降)
│  └──────────────────────────────┘    │
│  ▂▃▅▆▇██▇▆▅▃▂                    │ ← Sparkline 迷你趋势线
│  CPS-001 . 已连接 3421s             │ ← --font-mono, --text-xs, 元数据行
└──────────────────────────────────────┘
```

**CSS 约束**:
| 属性 | 值 |
|------|-----|
| 背景色 | `--color-white` (#FFFFFF) |
| 边框 | `--border-thick` (4px solid #000) |
| 内部装饰线 | 左侧 4px 红色竖线 (`--color-red`) |
| 阴影 | `--shadow-hard` (4px 4px 0 0 #000) |
| border-radius | **0** — 构成主义硬边，绝不使用圆角 |
| 内边距 | `--space-4` (16px) |
| 悬停态 | 背景变 `--color-grey-90`，阴影加深 2px |

**状态色映射**（卡片顶部 4px 横条或左侧 4px 竖线）:

| 状态 | 颜色令牌 | 含义场景 |
|------|---------|---------|
| ONLINE / OK | `--color-black` 或 `--color-green` | 正常运行、已连接 |
| WARNING | `--color-yellow` | 指标接近阈值、需关注 |
| CRITICAL | `--color-red` | 严重告警、需要立即处理 |
| OFFLINE | `--color-grey-50` | 连接断开、无数据 |

#### 4.3.2 卡片排列规则

- 同一行的卡片等宽等高（CSS Grid `1fr 1fr 1fr 1fr`）
- 逻辑关联的卡片之间间距 8px，不同组的卡片间距 24px（Gestalt 邻近性原则）
- 每个卡片内最多 1 个主指标 + 2 个辅助指标 + 1 条 sparkline
- 卡片自身不可滚动 —— 内容超长时截断或展开详情

### 4.4 间距令牌

```css
:root {
  --space-1: 4px;   /* 微间距 */
  --space-2: 8px;   /* 卡片间紧密间距 */
  --space-3: 12px;  /* 内边距 */
  --space-4: 16px;  /* 标准内边距 */
  --space-6: 24px;  /* 卡片间距 */
  --space-8: 32px;  /* 区域间距 */
  --space-12: 48px; /* 区块间距 */
  --space-16: 64px; /* 大区块间距 */
}
```

间距基于 4px 单位网格，所有元素（内边距、外边距、间距）必须是 4px 的整数倍。

## 5. 装饰元素

### 5.1 构成主义符号在看板中的映射

| 符号 | 构成主义含义 | 在看板中的用途 | CSS 实现 |
|------|-------------|---------------|---------|
| 红色楔形 (Red Wedge) | 革命力量 / 突破 | 异常/攻击状态可视化、卡片角标、图表数据标记 | `clip-path: polygon(0% 0%, 100% 0%, 0% 100%)` + `background: var(--color-red)` |
| 五角星 | 苏维埃/共产主义 | 标题栏左侧系统品牌标识、健康活跃标记 | 内联 SVG 符号，大小 16-24px |
| 齿轮 (Gear) | 工业化 / 工人阶级 | 设置按钮、刷新按钮、系统配置入口 | 内联 SVG 符号，大小 14-20px |
| 对角线切割 | 动力 / 张力 | 标题栏右下角切割、版本标签、页面区隔 | `clip-path: polygon(0 0, 100% 0, 100% 100%, 90% 100%, 100% 0)` |
| 嵌套方框 | 秩序 / 分层 | 卡片双边框效果、图表框架、组件容器 | `border: 4px solid #000` + `outline: 2px solid #CC0000` |
| 放射光芒 | 启蒙 / 未来 | 全屏数据刷新时的过渡背景、强调动画 | `conic-gradient()` 实现 |
| 粗横线分隔 | 力量 / 分割 | 区域之间的视觉分隔（标题栏下方、区域之间） | 8px 纯黑横线，无任何修饰 |

### 5.2 关键装饰 CSS 类

```css
/* 红色楔形覆盖（状态卡片异常角标） */
.wedge-corner {
  position: absolute;
  top: 0; right: 0;
  width: 48px; height: 48px;
  clip-path: polygon(100% 0, 100% 100%, 0 0);
  background: var(--color-red);
}

/* 对角线标题栏切割 */
.header-diagonal {
  clip-path: polygon(0 0, 100% 0, 100% 100%, calc(100% - 80px) 100%, 100% 0);
}

/* 嵌套方框容器 */
.construct-frame {
  border: 4px solid var(--color-black);
  outline: 2px solid var(--color-red);
  outline-offset: -8px;
}

/* 四角红色角标 */
.corner-accent {
  position: relative;
}
.corner-accent::before {
  content: '';
  position: absolute;
  top: 0; left: 0;
  width: 16px; height: 16px;
  border-top: 3px solid var(--color-red);
  border-left: 3px solid var(--color-red);
}
```

### 5.3 装饰使用原则

- **每个装饰必须有功能含义**：楔形 = 异常，五角星 = 健康，齿轮 = 可操作
- **装饰面积不超过页面总面积 10%**（标题栏 + 导航 + 卡片角标合计）
- **绝不使用"纯装饰"的图案**（没有功能含义的线条/形状一律禁止）
- **对角线只在关键位置使用**（标题栏、版本标签、异常角标），不要全局滥用

## 6. 数据可视化风格

### 6.1 图表配色

| 元素 | 颜色 | 规范 |
|------|------|------|
| 坐标轴 | `--color-black` | 2px 实线 |
| 网格线 | `--color-grey-90` | 1px 实线，仅水平方向，无垂直网格 |
| 主要数据线 | `--color-red` | 3px 实线，红色 = 功能色 |
| 次要数据线 | `--color-black` | 2px 实线（对比线、基线） |
| 柱状图填充 | `--color-black` | 纯色填充，柱顶红色装饰线 |
| 柱状图悬停 | `--color-red` | 悬停时柱体变红 |
| 填充区域 | `rgba(204, 0, 0, 0.08)` | 极淡红色半透明（折线下方） |
| 数据点标记 | `--color-red` | 实心圆，直径 6px，无阴影 |
| 十字准线 | `--color-red` | 红色虚线，悬停时显示 |
| 仪表盘/进度条 | `--color-black` 填充 | 超过阈值变为 `--color-red` |

### 6.2 图表类型与规范

| 图表类型 | 在看板中的用途 | 构成主义风格约束 |
|---------|--------------|-----------------|
| **折线图** | 时间序列指标（延迟、吞吐量、在线数） | 3px 红色粗线，无渐变填充，无图例（直接在端点标注数值） |
| **柱状图** | 多实体对比（CPS 健康分、各系统负载） | 黑色柱体 + 红色柱顶装饰线，柱宽 16-24px，柱间距 8px |
| **环形图** | 状态分布（在线/离线/告警占比） | 黑/红/黄 三色扇区，无渐变，标注百分比在扇区内 |
| **热力图** | 异常密度、时间段分布 | 红色单色渐变（从极浅红到纯红），色块无圆角 |
| **仪表盘** | 极值监控（CPU/内存接近阈值） | 半圆仪表盘，黑色刻度线，红色指针，无立体效果 |
| **时间线/甘特图** | 事件持续可视化 | 黑色时间条，告警事件条为红色，直接标注文字 |

### 6.3 实时数据渲染规范

| 策略 | 约束 |
|------|------|
| 增量渲染 | 只更新变化的数据点，不重绘整个图表 |
| 滑动窗口 | 固定显示最近 60s（默认），可切换 5min / 1h |
| 帧率锁定 | 30FPS 上限，使用 `requestAnimationFrame` 节流 |
| 动画曲线 | 仅使用 `steps(4, end)` 或 `cubic-bezier(1, 0, 0, 1)` — 拒绝 ease-in-out |
| 连接状态 | 图表右上角 8px 圆点：绿色=连接中，红色=断开，灰色=暂停 |
| 数据标签 | 图表内部直接标注数值，不使用悬浮图例 |

### 6.4 图表布局规则

- 图表标题左侧加 4px 红色竖线装饰
- 图表容器使用 `--border-thin` (1px solid #000) 边框
- 图表内文字全部使用 `--font-mono` + `--text-xs`
- 不支持 3D 图表、立体效果、阴影投影
- 图表背景透明（继承卡片背景）

## 7. 交互设计注意事项

### 7.1 构成主义风格的交互约束

| 交互 | 风格要求 | 说明 |
|------|---------|------|
| 悬停态 | 硬切换，无渐变过渡 | `transition: background 100ms steps(4, end)` |
| 点击态 | 瞬间视觉反馈 | 背景反转或边框变色，不使用缩放/弹跳 |
| 页面切换 | 硬切换（无滑动/淡入淡出） | 内容瞬间替换，或使用对角线擦除效果 |
| 数据刷新 | Steps 动画，不丝滑 | 数值变化时使用 steps(4) 跳变，缓入缓出与风格矛盾 |
| 弹窗/面板 | 从右侧滑入，硬阴影 | `--shadow-hard`，4px 黑色硬边框，0 圆角 |
| 拖拽排序 | 拖拽时显示硬边框指示器 | 虚线边框指示放置位置，不模糊/透明 |

### 7.2 可用性保障规则

虽然构成主义追求激进视觉，在看板场景中必须保证功能可用性，以下是不可妥协的可用性约束：

| # | 规则 | 说明 |
|---|------|------|
| 1 | **红色不能是唯一的信息传达方式** | 所有状态必须配合文字（如 "CRITICAL"）+ 形状（如倒三角），色盲用户也能识别 |
| 2 | **交互热区最小 44px** | 所有可点击元素（刷新按钮、卡片点击区、Tab）的交互区域 >= 44x44px |
| 3 | **字体对比度 >= 4.5:1** | `--color-grey-30` 在白色背景上对比度约 4.6:1，低于此不可用于正文 |
| 4 | **焦点指示器可见** | 键盘 Tab 导航时，焦点使用 2px 红色虚线框，清晰可见 |
| 5 | **暂停按钮始终可见** | 实时数据高频刷新时，用户需要随时暂停以详细查看，暂停按钮保持在标题栏右侧 |
| 6 | **数据暂停时有视觉提示** | 卡片/图表暂停刷新时，透明度降低 10% 或边框颜色变化，用户一眼可知 |
| 7 | **悬停信息触手可及** | 鼠标悬停在任意指标上，1.5 秒内显示 tooltip（带精确值 + 时间戳） |
| 8 | **移动端触控友好** | 卡片点击、滑动切换 Tab 等操作适配触控，tooltip 改为点击触发 |

### 7.3 钻取交互层级


## 8. 参考灵感

### 8.1 关键艺术家与作品

| 艺术家 | 生卒 | 在看板中的影响 | 代表作品 |
|--------|------|---------------|---------|
| **El Lissitzky** | 1890-1941 | 红色楔形符号、对角线构图、几何抽象语言 | 《Beat the Whites with the Red Wedge》(1919) — 红色楔形进入白色圆形的视觉隐喻 |
| **Alexander Rodchenko** | 1891-1956 | 极端对角线、photo-montage 技法、粗几何字体 | 《Books》海报(1924)，《Worker's Club》室内设计(1925)，Rodchenko 字体 |
| **Vladimir Stenberg / Georgii Stenberg** | 1899-1982 / 1900-1933 | 电影海报中的机械感构图、嵌套方框、粗轮廓线 | 《Battleship Potemkin》海报(1926)，《The Man with the Movie Camera》海报(1929) |
| **Gustav Klutsis** | 1895-1938 | photo-montage 与几何图形叠加、政治宣传视觉 | 《The USSR is the Shock Brigade of the Proletariat of the World》(1930) |
| **Varvara Stepanova** | 1894-1958 | 功能性服装/纺织设计、网格布局、实用主义美学 | 《Sports Clothing》设计(1923) |

### 8.2 设计话语中的关键概念

- **"形式追随功能" (Form Follows Function)** — 构成主义的信条，看板的每个元素都必须传达信息，拒绝为装饰而装饰
- **"Tectonics" (构造性)** — 材料与结构的诚实表达，对应看板中数据原始状态的直接展示
- **"Factography" (事实摄影)** — Rodchenko 倡导的纪实手法，对应看板中数据"不加修饰地呈现"
- **"Agitprop" (宣传鼓动)** — 视觉的动员力量，对应告警/异常需要引起注意的设计

### 8.3 致敬参考的经典海报构图

**Beating the Whites with the Red Wedge (Lissitzky, 1919)**:
- 红色楔形突破白色圆形 = 异常数据突破阈值范围的可视化
- 几何体直接作为叙事语言 = 数据形状本身传达含义

**Rodchenko's Books Poster (1924)**:
- 极端对角线排版 = 标题栏右下角切割
- 巨大字号对比 = KPI 数值与标签的 4.6 倍字号差
- 黑白摄影 + 纯色几何 = 数据图表 + 构成主义装饰

**Stenberg Brothers' Film Posters (1920s)**:
- 嵌套方框 + 机械感构图 = 卡片双边框设计
- 粗轮廓线 + 纯色填充 = 图表数据呈现方式
- 对角线 + 圆形切割 = 页面布局中的视觉张力

## 附录 B: 反模式清单（开发检查用）

前端开发在实现中应避免以下设计模式，偏离时将破坏风格一致性：

1. `border-radius > 0` 的任何元素（包括卡片、按钮、输入框、图表柱体）
2. `box-shadow` 模糊半径 > 0（仅允许硬阴影）
3. `transition: all 0.3s ease` 等缓入缓出动画（替换为 `steps(4)`）
4. 渐变背景（`linear-gradient`、`radial-gradient` 等）
5. 小写标题（卡片标题、导航、标签必须 `text-transform: uppercase`）
6. 超过 3 种颜色的图表（限制为红、黑、灰三色）
7. 浮动操作按钮（FAB）或悬浮元素
8. 模态弹窗（使用侧面板或叠加层替代）
9. 阴影/立体效果的数据图表
10. 自动轮播或页面自动切换焦点
11. SVG 图标自带颜色（图标应继承 `currentColor` 以适应红/黑切换）
12. 任何形式的发光/霓虹效果（`text-shadow`、`filter: glow` 等）
