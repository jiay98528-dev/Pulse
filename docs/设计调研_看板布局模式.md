## 二、核心设计原则

### 2.1 看板设计六大定律（适配 Pulse）

| 定律 | 原理 | Pulse 应用 |
|------|------|-----------|
| **Fitts 定律** | 大+近=快找到 | 关键指标左上角，最大字号，零定位成本 |
| **Gestalt 邻近性** | 空间接近=逻辑关联 | 同一数据源的卡片间距更小，形成视觉组 |
| **Miller 7±2** | 工作记忆有限 | 每视图聚焦一个核心问题，KPI 不超过 6 个 |
| **Hick 定律** | 选择越少决策越快 | 渐进式披露：先摘要再钻取 |
| **构成主义铁律** | 红黑白三色、全大写、硬边 | 所有视觉元素遵循构成主义令牌 |
| **5 秒法则** | 关键信息 5 秒内可识别 | 数据层级清晰，一眼看到健康状态 |

### 2.2 构成主义 × 数据看板的融合策略

构成主义原本就是"功能性优先"的设计运动——形式追随功能，每个元素必须传达信息。这与实时监控看板的目标天然一致。

融合策略：
- **色彩**：保留红/黑/白为主色，用红色作为关键告警/异常锚点，黑色为背景，白色为数据文字
- **几何语言**：用红色楔形(三角形)表示"异常/攻击"状态，硬边框框定数据卡片区域
- **排版**：标题全部大写，使用 Tektur/Major Mono Display 等构成主义字体；数据正文用 JetBrains Mono 等宽字体保持技术感
- **对角线**：在导航栏/标题栏使用对角线切割作为视觉区隔，而非柔和分隔线
- **硬边**：拒绝圆角、阴影、渐变——所有卡片 border-radius: 0，使用硬阴影(4px 偏移)

```
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐          │
│  │ CPS #1  │ │ CPS #2  │ │ CPS #3  │ │ CPS #4  │          │  ← 状态总览卡片行 (1×4)
│  │ ● 在线  │ │ ● 在线  │ │ ● 正常  │ │ ● 告警  │          │     每个卡片一个数据源

│  └─────────────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────────────┘
```

### 3.2 Z 型扫描路径

用户视线自然地从左到右、从上到下扫描，因此关键信息必须有层级：

| 区域 | 内容 | 视觉权重 |
|------|------|---------|
| **Zone 1 (左上)** | 标题栏 + 全局状态 | 最大 (红色背景+白色大字) |
| **Zone 2 (右上)** | CPS 状态卡片行 | 次大 (硬边框+构成主义符号) |
| **Zone 3 (中左)** | 主要趋势图表 | 中 (白底黑线+红强调) |
| **Zone 4 (中右)** | 辅助对比图表 | 中 |
| **Zone 5 (底部)** | 事件流日志 | 小 (等宽字体, 低视觉权重) |

### 3.3 响应式断点

| 断点 | 列数 | 布局变化 |
|------|------|---------|
| ≥1400px | 12列 | 标准布局, 2列图表+4列卡片 |
| 960-1400px | 8列 | 图表改为单列堆叠, 卡片2×2 |
| 600-960px | 4列 | 全部单列, 收起事件流为可展开 |
| <600px | 2列 | 仅显示 KPI 卡片+1个主要图表 |

## 四、UI 组件设计规范

### 4.1 卡片 (Card)

**设计依据**: 构成主义嵌套方框 + 监控看板 KPI 卡片模式

```
┌──────────────────────────────┐
│  ▐  STATUS: ACTIVE           │ ← 标题行: 全部大写, Tektur 字体
│  ┌──────────────────┐       │   红色竖线(2px) 左侧装饰
│  │            98.5%  │       │ ← 主指标值: 巨大字号, 等宽字体
│  │            ▲ 2.1% │       │   趋势方向 + 变化百分比
│  └──────────────────┘       │
│  ▂▃▅▆▇██▇▆▅▃▂           │ ← 迷你趋势线 (Sparkline)
│  CPS-001 · 已连接 3421s     │ ← 元数据行
└──────────────────────────────┘
```

**令牌映射**:
| 属性 | 令牌值 |
|------|--------|
| 背景色 | `--color-white` (#FFFFFF) |
| 边框 | `--border-thick` (4px solid #000) |
| 内边框 | `--outline-construct` (2px solid #CC0000) |
| 阴影 | `--shadow-hard` (4px 4px 0px 0px #000) |
| 标题字体 | `--font-display` |
| 数据字体 | `--font-mono` |
| 标题字号 | `--text-xs` (0.75rem) 全大写 |
| 指标字号 | `--text-4xl` 到 `--text-5xl` |
| 间距 | `--space-4` 内边距 |
| 角半径 | `0` (硬边) |
| 悬停态 | 背景变 `--color-grey-90`, 阴影加深 |

**状态色映射** (卡片左边框/顶部条):
| 状态 | 颜色 |
|------|------|
| 正常 (OK) | `--color-black` |
| 告警 (Warning) | `--color-yellow` |
| 严重 (Critical) | `--color-red` |
| 离线 (Offline) | `--color-grey-50` |

**交互规则**:
- 悬停: 背景变为极浅灰, 显示"查看详情"指示
- 点击: 展开侧面板显示该数据源的详细指标 (渐进式披露)
- 双击: 进入该数据源的专属详情页面 (钻取)

### 4.2 图表 (Chart)

**实时数据的核心挑战**: 每秒刷新时不能全量重绘，必须使用增量渲染策略。

#### 4.2.1 图表类型选择矩阵

| 数据类型 | 推荐图表 | 原因 | 实时适配策略 |
|---------|---------|------|------------|
| 时间序列指标 (延迟/吞吐量) | **折线图** | 线段局部重绘, 性能最优 | 滑动窗口, 追加新点移除旧点 |
| 多实体对比 (CPS 健康分) | **柱状图** | 横向对比一目了然 | 仅更新数值, 柱高动画过渡 (100ms steps) |
| 状态分布 (在线/离线/告警) | **环形图** | 比例感直观 | 扇区角度动态微调, 不重绘 |
| 异常密度 | **热力图** | 快速定位异常聚集区 | 色块无闪烁局部更新 |
| 复合指标 | **组合图** (柱+折线) | 同时看值和趋势 | 柱状增量+折线追加 |
| 极值监控 | **仪表盘/进度条** | 阈值直观 | 指针/填充动画, 无数据刷新 |
| 事件时间线 | **甘特图/时间线** | 事件持续可视化 | 实时追加事件条 |

#### 4.2.2 构成主义风格图表规范

| 属性 | 规范 |
|------|------|
| 坐标轴颜色 | `--color-black` (2px 硬线) |
| 网格线 | `--border-thin` (1px solid #E6E6E6), 仅水平网格 |
| 折线颜色 | `--color-red` (#CC0000) — 红色是数据线 |
| 折线宽度 | 3px — 粗线构成工业感 |
| 柱状颜色 | `--color-black` 填充, 柱顶红色装饰 |
| 填充区域 | `rgba(204, 0, 0, 0.1)` 极淡红透明 |
| 数据点标记 | 红色实心圆 (6px), 无阴影 |
| 数据标签 | `--font-mono`, `--text-xs`, 直接标注(不依赖图例) |
| 背景 | `--color-white` 或 `--color-grey-10` (黑底时白色折线) |
| 图例 | 无图例, 直接在图表内部标注 (节省 40% 处理时间) |
| 十字准线 | 红色虚线, 悬停时显示同步值 (跨图表联动) |
| 动画 | `--ease-mechanical` steps(4), 100-200ms 硬过渡 |

#### 4.2.3 实时数据刷新策略

| 策略 | 实现 |
|------|------|
| **增量渲染** | 只更新变动的数据点, 不重绘整个图表 |
| **滑动窗口** | 固定显示最近 60s/300s/3600s, 新数据追加, 旧数据滑出 |
| **帧率锁定 30FPS** | 浏览器 requestAnimationFrame 节流, 避免过度渲染 |
| **智能降级** | 当每秒变化点数 > 阈值时, 暂缓动画, 直接跳转最新值 |
| **WebSocket 推送** | 服务端主动推送差异数据包 (<1KB), 而非轮询 |
| **连接状态指示** | 图表右上角小圆点: 绿=连接中, 红=断开, 点击可手动刷新 |

### 4.3 标题栏 (Header / Title Bar)

**设计依据**: 构成主义极端的红黑对比 + 对角线切割 + 放大字号

```
┌──────────────────────────────────────────────────────────────┐
│  ★  PULSE·MONITOR                    ● ONLINE  |  设置   │
│  ──────────────────────────────────────────────────          │
│  SYSTEM STATUS: ALL SYSTEMS NOMINAL  |  2026-06-19 14:30   │
└──────────────────────────────────────────────────────────────┘
```

| 元素 | 规范 |
|------|------|
| 背景 | `--color-red` (#CC0000) 或 `--color-black` |
| 文字颜色 | `--color-white` (反白) |
| 主标题字体 | `--font-display`, `--text-3xl` (40px), 全大写 |
| 副标题字体 | `--font-mono`, `--text-sm`, 全大写 |
| 字母间距 | `--tracking-wider` (0.2em) |
| 标题左侧装饰 | 五角星 SVG (14px, 白色或金色) |
| 右下对角线切割 | `clip-path: polygon(0 0, 100% 0, 100% 100%, 90% 100%, 100% 0)` |
| 分隔线 | `--divider-thick` (8px 黑线) |
| 状态圆点 | 直径12px, 实心圆, 绿色/红色 (脉冲动画) |

**左/右分区**:
- 左侧: 项目名 + 标语 + 星装饰 + 系统时间
- 右侧: 全局连接状态 + 刷新按钮(齿轮图标) + 设置齿轮

### 4.4 状态指示器 (Status Indicator)

**构成主义符号系统 + 实时监控状态指示**

#### 4.4.1 三种状态指示器

```
类型 A: 圆点指示器 (紧凑, 用于标题栏/列表)
  ● 已连接 (绿色实心, 6ms 呼吸动画)
  ◌ 连接中 (黄色, 闪烁动画, steps(4))
  ○ 已断开 (灰色, 静态)

类型 B: 状态标签 (用于卡片标题)
  ┌──────────┐
  │ ACTIVE   │  ← 黑底白字, 4px硬边框
  └──────────┘
  ┌──────────┐
  │ WARNING  │  ← 红底白字 (或黄底黑字)
  └──────────┘

类型 C: 构成主义几何指示器 (特色, 用于全屏概览)
  ▲  正常   (黑色正三角 = 稳定)
  ▼  异常   (红色倒三角 = 警示)
  ◆  维护中 (灰色菱形)
```

#### 4.4.2 数据新鲜度指示器

每个数据卡片右下角显示数据新鲜度, 用颜色编码:

| 新鲜度 | 颜色 | 显示 |
|--------|------|------|
| <3s | `--color-green` | "LIVE" 或 "实时" |
| 3-10s | `--color-black` | "2s ago" |
| 10-30s | `--color-yellow` | "15s ago" |
| >30s | `--color-red` | "STALE" + 手动刷新按钮 |

## 五、交互方式规范

### 5.1 悬停交互 (Hover)

| 场景 | 行为 | 动效 |
|------|------|------|
| 卡片悬停 | 背景变灰 `--color-grey-90`, 右侧出现 "→" 箭头 | 100ms, no easing |
| 折线图悬停 | 红色十字准线, 数据点放大, tooltip 显示精确值 | 200ms steps(4) |
| 柱状图悬停 | 柱体颜色变红, 柱顶显示数值标签 | 100ms steps(4) |
| 状态指示器悬停 | tooltip 显示详细状态和最近更新时间 | 100ms |
| 导航项悬停 | 背景变深灰, 文字轻微左移 (skewX -3deg) | 100ms |

### 5.2 点击交互 (Click)

| 场景 | 行为 |
|------|------|
| 卡片点击 | 展开侧面板 (从右侧滑入, 400px宽), 显示该数据源的详细指标 |
| 图表数据点点击 | 在该图表上方叠加详情浮层, 或钻取到二级详情页 |
| 状态指示器点击 | 展开事件时间线, 聚焦到该状态的最近事件 |
| 事件流条目点击 | 在页面中央展开事件详情浮层 |
| 导航 Tab 切换 | 页面内容过渡切换, 无滑动/淡入淡出 (硬切换) |

### 5.3 钻取层级 (Drill-Down)

```
Level 1: 总览看板 (当前页面)
   ├→ Level 2: 数据源详情 (侧面板展开)
   │    ├→ Level 3: 单个指标历史趋势 (弹窗/浮层)
   │    └→ Level 3: 关联事件列表 (底部面板展开)
   └→ Level 2: 全屏图表 (点击图表右上角全屏图标)
```

- 每个钻取步骤必须有"返回"按钮 (构成主义 ← 符号)
- 钻取路径标记在面包屑中: `PULSE > CPS-004 > 延迟分析`

### 5.4 实时控制

| 控制 | 行为 |
|------|------|
| 暂停/恢复 | 冻结所有实时更新, 用于仔细查看数据 |
| 时间回溯 | 滑块或按钮组: 30s / 5min / 1h / 24h |
| 全屏模式 | 隐藏侧边栏和标题栏, 仅显示图表区 |
| 手动刷新 | 点击卡片上的刷新按钮 (齿轮图标) |

## 六、构成主义视觉系统在看板中的映射

| 构成主义元素 | 看板应用 | 具体实现 |
|------------|---------|---------|
| 红色楔形 (Lissitzky) | 异常/攻击状态的图形化表达 | `clip-path: polygon(0% 0%, 100% 0%, 0% 100%)` 叠加在异常卡片上 |
| 五角星 | 系统"健康/活跃"标记 | 标题栏左侧, 或卡片右上角标记活跃状态 |
| 齿轮 | 系统/配置入口 | 设置按钮, 刷新按钮 |
| 对角线 | 页面视觉区隔 | 标题栏右下角切割, 状态卡片之间的斜线分隔 |
| 嵌套方框 | 卡片边框装饰 | `border-construct` 类: 4px 红边 + 2px 黑内轮廓 |
| 放大缩小对比 | KPI 数值 vs 标签 | KPI 数值 3.5rem, 标签 0.75rem (4.6倍差距) |
| 粗轮廓线 | 卡片边界、图表坐标轴 | 均为 4px 或 2px 实线 |
| 全部大写 | 标题、标签、导航 | 所有文字标签 ALL CAPS |
| 等宽字体 | 技术数据展示 | 所有数值使用 JetBrains Mono |
| 硬阴影 | 卡片、弹窗 | `4px 4px 0px 0px #000` |

## 七、推荐技术栈

| 层 | 推荐 | 原因 |
|----|------|------|
| 图表库 | **ECharts** (增量渲染 + WebSocket) | 原生支持增量更新、大数据集、帧率控制 |
| 备选图表 | **uPlot** | 极致性能 (可支撑 100万+ 数据点) |
| 布局 | **CSS Grid** + `react-grid-layout` (如需拖拽) | 原生网格, 兼容构成主义正交网格 |
| 实时通信 | **WebSocket** | 服务端主动推送, 增量数据包 |
| 动效 | CSS `steps()` 或 WAAPI | 无缓入缓出, 机械感硬过渡 |
| 状态管理 | Zustand 或 Signals | 轻量、高性能、适合高频更新 |
| 图标 | 内联 SVG (构成主义符号集) | 无网络请求, 可精确控制颜色 |

## 八、反模式清单 (Avoid)

以下是不适合 Pulse 的设计模式, 源自行业经验和构成主义约束:

1. **柔和圆角卡片** (`border-radius > 0`) —— 违反构成主义硬边原则
2. **渐变色图表** —— 只用纯色填充
3. **柔滑动画** (ease-in-out) —— 必须使用 steps() 硬过渡
4. **多色彩图表** (>3色) —— 只用红、黑、灰, 必要时加黄色警示
5. **浮动操作按钮 (FAB)** —— 构成主义没有"漂浮"元素, 所有操作紧贴内容
6. **全屏刷新加载动画** —— 使用骨架屏 + 局部增量更新, 不打断用户
7. **模态弹窗** —— 使用侧面板或叠加层代替, 保持上下文
8. **数据标签悬浮图例** —— 直接在图表中标注, 节省用户视线移动
9. **自动轮播/切换焦点** —— 用户必须完全控制看板焦点, 不要自动切换
10. **超过 30FPS 的刷新** —— 锁定 30FPS, 超过时智能降级
