# GaugePane v3 需求变更：欢迎页、主题/插件一级入口与发行门禁

> 日期：2026-06-24  
> 状态：当前 v3 需求权威补丁  
> 品牌口径：对外品牌为 GaugePane / 仪窗；代码库与部分历史文档短期仍保留 Pulse 命名。  
> 适用范围：本文件覆盖当前 v3 需求，不修改 `docs/需求规格说明书_v2.0.md`。v2.0 文档已是 LEGACY，仅作历史参考。

---

## 1. 变更摘要

本轮需求变更解决四个产品发现性与发行门禁问题：

1. 增加首次欢迎页，让新用户理解 GaugePane 的核心能力和配置路径。
2. 将“主题”和“插件”提升为一级分栏，避免商业化和扩展能力被藏在设置页深处。
3. 将内置主题市场列为发行门禁，Preview 版本允许在线服务离线，但必须能本地切换主题并展示市场结构。
4. 发行前至少预置 5 个完全不同的官方主题，旧 legacy 主题不能计入。

---

## 2. 欢迎页需求

### 2.1 触发规则

- 首次启动自动弹出欢迎页。
- 用户点击“完成”或“稍后再说”后，本版本不再自动弹出。
- 设置页必须保留“重新打开欢迎页”入口。
- 状态键固定为 `gaugepane-welcome-v1-seen`。
- 实现时可兼容历史 Pulse 命名的本地状态键，但对外 UI 不展示旧品牌。

### 2.2 交互边界

- 欢迎页可以是 overlay、modal 或独立 page，但不得成为长期阻塞遮罩。
- 用户必须可以跳过。
- 跳过欢迎页不应伪造配置完成状态。
- 未配置 AI、未启用 LAN、主题市场离线时，欢迎页只能解释和引导，不得隐藏功能入口。

### 2.3 内容模块

欢迎页至少包含 5 个模块：

| 模块 | 必须说明 |
|---|---|
| 本地系统仪表 | CPU、内存、磁盘、网络、电池和运行状态可在本机实时查看 |
| AI 监看与配置 | DeepSeek 当前为 active；OpenAI/Anthropic 可保存配置但不等同于已接入遥测 |
| Codex 状态 | 只检测本机 CLI/登录状态，额度引导到官方 Codex 用量入口 |
| 局域网多设备监看 | 通过插件启用 LAN 发现、配对和共享指标 |
| 主题市场与主题编辑 | 本地主题可切换，在线市场离线时仍可使用本地主题 |

### 2.4 固定 CTA

欢迎页 CTA 固定为：

- 进入仪表盘
- 配置 AI
- 打开主题
- 查看插件
- 稍后再说

CTA 行为必须使用现有 tab 路由机制，不刷新页面，不绕过 `activateTab()`。

## 3. 一级导航需求

### 3.1 导航结构

新一级导航固定为：

```text
仪表盘 / 硬件 / 分析 / 主题 / 插件 / 设置
```

### 3.2 主题页

“主题”必须从设置页提升为一级页面，承载：

- 当前主题选择与本地切换。
- 内置主题市场入口与筛选。
- 主题详情、预览、安装、购买或清晰失败态。
- `.pulse-theme` / 后续 `.gaugepane-theme` 导入导出。
- 主题编辑器。
- 本地主题热更新状态。

设置页可以保留到主题页的跳转提示，但不得再作为主题功能唯一入口。

### 3.3 插件页

“插件”保持一级页面，并且必须在欢迎页和设置页中可见：

- 插件列表和启停状态。
- LAN 插件配置、发现、配对和共享指标入口。
- 插件不可用或未启用时显示解释性空态。

插件不得被当作隐藏高级功能，也不得只在某个条件满足后才出现在导航中。

### 3.4 设置页

设置页保留：

- AI Provider / API Platform Key / DeepSeek 配置。
- Codex 本机状态摘要和官方用量入口。
- 开机自启与通用设置。
- 关于与版本信息。
- 欢迎页重开入口。

主题相关功能不得只存在于设置页深处。

---

## 4. 主题市场发行门禁

### 4.1 市场形态

内置主题市场默认形态为：

- 主题一级页面中的市场区域。
- 主题详情弹窗或详情面板。
- 可选的全屏/大弹窗浏览模式。

主题市场不得只依赖设置页小卡片。

### 4.2 筛选能力

必须支持筛选：

- 全部
- 官方
- 社区
- 本地
- 已安装

Preview 版本至少支持：官方、社区、本地。

### 4.3 Preview 门禁

Preview 版本允许没有真实下载服务器和支付闭环，但必须满足：

- 能打开内置主题市场窗口或主题页。
- 能切换本地预置主题。
- 能展示官方/社区/本地分类。
- 在线市场 8081 未运行时显示“在线市场离线，本地主题仍可使用”。
- 主题详情或卡片不能空白，按钮不能静默失败。

### 4.4 Stable 门禁

Stable 版本必须额外满足：

- 在线主题列表可加载。
- 官方/社区筛选可用。
- 主题详情可打开。
- 下载或购买链路可用，或给出明确失败态。
- 恢复已购可用，或给出明确失败态。
- 支付、恢复、下载相关错误不得表现为空白市场。

## 5. 官方主题门禁

### 5.1 数量与标准

发行前至少预置 5 个完全不同的官方主题：

| ID | 中文名 | 定位 |
|---|---|---|
| `gauge-ops` | 仪窗默认 | 默认深色遥测仪表 |
| `frost-console` | 霜白控制台 | 浅色/冷色低噪工作台 |
| `amber-terminal` | 琥珀终端 | 复古仪表与终端风格 |
| `graphite-studio` | 石墨工位 | 极简专业工作台 |
| `aurora-desk` | 极光桌面 | 更适合桌搭传播的彩色主题 |

### 5.2 元数据要求

每个官方主题必须包含：

- `schemaVersion: 3`
- `id`
- `name`
- `author`
- `version`
- `type: official`
- 预览元数据
- 完整 v3 tokens
- 可读性说明

### 5.3 不计入门禁的主题

- `constructivist` 只能作为 legacy 兼容主题，不计入 5 个官方主题。
- 仅修改名称、未形成明显视觉差异的变体不计入。
- 无完整 v3 tokens 的旧主题不计入。

## 6. 验收标准

### 6.1 文档验收

- PRODUCT、UI 重构规格、TODO、交接文档、全景总结、错题本必须同步本需求。
- 商业计划书不得继续声称发行前只需 3 个官方主题。
- Legacy v2 需求规格不作为本轮修改目标。

### 6.2 实现验收口径

后续实现必须满足：

- 首次欢迎页可跳过、可重开，不是长期遮罩。
- URL tab 与点击 tab 都支持主题、插件、设置一级导航。
- 主题一级页面可在 960x600 和 1280x800 下无横向溢出。
- Preview 门禁下，在线主题市场离线也不影响本地主题切换。
- Stable 门禁下，在线主题下载或购买链路必须真实可用或有明确失败态。
- 至少 5 个官方主题可本地切换，且 legacy constructivist 不计入。

### 6.3 禁止事项

- 不得把欢迎页做成无法关闭的全局遮罩。
- 不得把主题功能重新藏回设置页深处。
- 不得把插件隐藏成只有启用后才可见的高级项。
- 不得让在线主题市场离线时显示空白。
- 不得把旧 legacy 主题算作官方主题门禁。
