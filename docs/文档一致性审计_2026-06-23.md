# Pulse 文档一致性审计（2026-06-23）

## 结论

当前项目没有发现权威需求文档损坏或编码腐败。此前终端中出现的乱码来自 PowerShell 输出编码，不是 Markdown 文件本身损坏。

实际实现与当前权威需求链总体一致：

- 权威链：`PRODUCT.md`、`DESIGN.md`、`docs/UI_REFACTOR_SPEC_GPT5_CODEX_SPARK.md`、`docs/错题本.md`。
- 实现链：`frontend/index.html`、`frontend/js/app.js`、`frontend/js/telemetry-canvas.js`、`frontend/css/style.css`。
- 后端 API 未被删除，当前 v3 修复集中在前端运行时、可见性、空态和验收路径。

## 已发现并修正的偏移

| 偏移 | 影响 | 当前状态 |
|---|---|---|
| Dashboard 新增 6 个 Canvas 曾只有 DOM、无 controller | 首页主视觉空白，像功能丢失 | 已接入 `DashboardTelemetryController` |
| URL `?tab=` / hash 曾不参与初始化 | 直达验收路径误判页面丢失 | 已统一到 `activateTab()` |
| 首次设置曾是全局阻塞 overlay | 设置/插件/硬件页被遮挡 | 已改为非阻塞 `setup-banner` |
| AI widgets 未配置时曾被自动删除 | 用户误以为余额/Token/缓存功能被删 | 已改为锁定空态 |
| 主题市场依赖 8081 但失败态不清晰 | 市场像空白或按钮失效 | 已增加明确离线态 |
| 隐藏 tab / shim 图表可能保持 1x1 | 硬件 CPU 负载动画丢失，启动卡顿 | 已让隐藏 Canvas 跳过 RAF，并让 shim canvas 填满容器 |
| 设置页关于卡片仍显示 Chart.js / 构成主义 | UI 文案与实际技术栈冲突 | 已改为 TelemetryCanvas / AI Telemetry |
