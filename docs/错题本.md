## 设计约束总结（Checklist）

1. **不简陋** — 每个几何块都是有意安排的精致构图，不是省事的产物
2. **不浮夸** — 无渐变、无发光、无毛玻璃、无圆滑边框，硬边直角是核心语言
3. **不赛博** — 这是1920s的工厂美学，不是2049年的夜店美学
4. **红色永远不作大面积背景** — 仅用于标题栏横条、数据线、告警态、几何装饰
5. **黑底白字或白底黑字二选一** — 不混合使用
6. **图表最多三色** — 红、黑、灰，必要时加黄色以示警告
7. **直角优先** — border-radius: 0 是构成主义的核心标识
8. **硬阴影** — box-shadow 仅有偏移量无模糊半径
9. **Surface Go 远距可读** — 最小字号 14px，KPI 数值 56px
10. **触屏热区 ≥ 44px** — Tab 按钮、卡片点击、表单交互

## 已排除的设计方向

| 方向 | 排除原因 |
|------|---------|
| 浅色/明亮背景 | 与构成主义的工业黑暗美学冲突，且小屏远距观看时高亮背景增加视觉疲劳 |
| 圆角卡片/组件 | 构成主义的核心是硬边几何，圆角软化革命能量 |
| 渐变图表填充 | 纯色不加渐变，保持宣传画般的坚实质感 |
| 霓虹/发光效果 | 赛博朋克风格与本项目定位冲突 |
| 侧边栏导航 | Surface Go 屏幕宽度有限，顶栏 Tab 占用更少空间 |
| 多级菜单 | 一级 Tab 导航已足够，钻取通过点击卡片展开 |

## 2026-06-20 Deepseek API 审查结论

### 审查摘要

对 [Deepseek 公开 API 文档](https://api-docs.deepseek.com) 与 Pulse 采集器进行了一致性审查。

**官方端点清单（仅 4 个）：**
| 端点 | 用途 | Pulse 使用 |
|------|------|-----------|
| `POST /chat/completions` | LLM 对话 | ❌ 不在采集范围 |
| `POST /beta/completions` | FIM 代码补全 | ❌ |
| `GET /models` | 列出模型 | ❌ |
| `GET /user/balance` | 查询余额 | ✅ 已使用 |

**审查结论：Deepseek 没有提供公开的 Token 用量查询 API。** 不存在 `/dashboard/usage`、`/v1/usage` 或任何聚合用量端点。

### 架构影响

当前 `backend/collectors/deepseek.py:67` 对 `GET /dashboard/usage` 的调用是对 Deepseek Web 控制台内部接口的依赖，**没有 SLA 保障**。正确的数据获取链路应是：

```
chat/completions 响应.usage
    → 调用方拦截
    → POST /api/deepseek/log (Pulse 本地端点)
    → SQLite deepseek_usage 表
    → Chart.js 历史展示
```

### /user/balance 响应完整 schema（用于后续修复参考）

```json
{
  "is_available": true,
  "balance_infos": [{
    "currency": "CNY",
    "total_balance": "110.00",       // 当前只取了这一个
    "granted_balance": "10.00",      // ⚠ 缺失 — 赠金，优先扣减，有有效期
    "topped_up_balance": "100.00"    // ⚠ 缺失 — 充值，永不过期
  }]
}
```
