# Pulse 错题本

> 本文件记录当前有效的设计/开发约束，以及已经废弃的历史约束。后续 agent 必须优先读取本文件，避免把 legacy 资料误当成当前规范。

## 当前有效 Checklist

1. **不回到旧构成主义**：红黑白、苏维埃符号、硬边直角、禁止渐变/发光/圆角等旧规则已经废弃。
2. **不做普通 SaaS 仪表盘**：避免同尺寸卡片墙、hero metric 模板、蓝紫渐变套壳。
3. **动效必须服务状态**：动画只表达数据刷新、趋势、阈值、在线/离线、用户操作反馈。
4. **远距可读**：正文最小 14px，关键 KPI 48px 以上。
5. **触屏可用**：所有可点击控件热区不小于 44px。
6. **性能优先**：Surface Go 默认 24-30fps，隐藏页面暂停动画，历史点有上限。
7. **主题可迁移**：旧主题必须兼容迁移到 schema v3，不能让用户配置失效。
8. **后端 API 不破坏**：UI 重构不改变 `/ws`、`/api/config`、`/api/system/current`、`/api/analysis/*` 等接口。
9. **离线韧性**：AI 数据失败时仍展示本地系统数据和缓存状态。
10. **文档无污染**：当前权威文档不得继续包含未标记为 legacy 的旧视觉禁令。
11. **Canvas 可见性正确**：隐藏 tab 的 Canvas 不参与 RAF，不允许以 1x1 作为最终布局。
12. **空态不等于消失**：未配置 API key、主题商店离线等情况必须显示解释性空态，不能静默隐藏功能。
13. **欢迎页不能变遮罩**：首次欢迎页必须可跳过、可重开，不能长期阻塞所有页面。
14. **主题 / 插件必须可发现**：主题和插件是一级功能入口，不能藏在设置页深处，也不能隐藏成高级项。
15. **主题发行门禁**：Preview 可无在线下载服务器，但本地主题切换和市场分类必须可用；Stable 才要求下载/购买闭环。

## Legacy 约束记录

以下内容只作为历史记录，不再约束当前实现：

| Legacy 方向 | 废弃原因 |
|---|---|
| 苏维埃构成主义作为唯一品牌方向 | 用户已确认彻底重置为 AI Telemetry / Next-gen Operations Surface |
| 红/黑/白三色系统占 90%+ | 限制动态图表和多状态遥测表达 |
| 红色永远不作大面积背景 | 属于旧视觉系统特例，新系统使用语义 signal token |
| 无渐变、无发光、无圆角 | 与次世代动态监控目标冲突，改为“视觉效果必须服务数据状态” |
| Chart.js 作为唯一图表层 | 已替换为自研 Canvas telemetry engine |
| 自由 widget 墙首页 | 改为半固定可配置骨架，避免首屏无重点 |

## 2026-06-24 GaugePane 欢迎页与主题发行门禁

问题：主题和插件虽然已有功能基础，但主题入口长期在设置页深处，插件也容易被理解成高级/可忽略功能；新用户首次启动缺少功能总览，难以理解 AI 配置、LAN 多设备监看和主题市场的商业化价值。

根因：
- 旧信息架构把主题作为设置项，而不是一级产品能力。
- 首次设置经历过全局遮罩问题，后续只改成非阻塞 banner，没有形成完整欢迎页。
- 主题市场曾作为设置卡片或弹窗存在，但没有被定义成发行门禁。

教训：
- 欢迎页必须解释功能和配置路径，但不得回到无法关闭的全局遮罩。
- 主题 / 插件必须保持一级可见入口，不能藏在设置页深处。
- Preview 与 Stable 发行门禁要分开：Preview 可只验证本地主题切换和市场离线态，Stable 才要求在线下载/购买/恢复闭环。
- 发行前至少 5 个官方 schema v3 主题可本地切换，legacy constructivist 只能算兼容主题，不能计入门禁。

排除清单：不得把欢迎页做成长期全局遮罩；不得把主题功能重新放回设置页深处作为唯一入口；不得把插件隐藏成只有启用后才出现的高级项；不得让主题市场离线时空白；不得把 legacy constructivist 算入 5 个官方主题。

实施状态：
- 已新增 `gaugepane-welcome-v1-seen` 欢迎页流程，设置页可重新打开。
- 已新增“主题”一级页面，承载主题切换、市场筛选、导入导出、编辑器和热更新状态。
- 已预置 `gauge-ops`、`frost-console`、`amber-terminal`、`graphite-studio`、`aurora-desk` 五个官方 schema v3 主题。
- Preview 门禁以本地主题切换和“在线市场离线，本地主题仍可使用”离线态为准；Stable 仍需在线下载/购买/恢复闭环。

## 2026-06-24 Codex 额度与主题热更新

问题：测试中把 OpenAI API Key 误认为可查询 ChatGPT/Codex 订阅剩余额度，同时本地主题文件修改后需要刷新页面或重新导入才生效。

根因：
- OpenAI API Platform Key 与 ChatGPT/Codex 订阅权益不是同一套查询口径；Codex 额度应在 Codex Settings > Usage 或活跃 CLI 会话 `/status` 查看。
- 主题系统已有导入、安装、激活和本地主题发现，但没有对 `frontend/themes/<id>/theme.json` / `custom.css` 做轮询刷新。

教训：
- 不得把 Codex 订阅额度伪装为 OpenAI API Key 查询结果；本机联动只能返回 CLI 是否存在、登录是否配置等红acted 状态。
- 纯 Web 文件输入不能长期监听任意外部文件；当前只支持 `frontend/themes` 本地主题轮询热更新，外部 `.pulse-theme` 仍需重新拖入安装。

排除清单：不得读取 `~/.codex/auth.json`；不得抓取私有 ChatGPT/Codex 接口；不得把 API Platform 用量文案写成 Codex 订阅额度；不得为热更新创建多个定时器或隐藏页面持续轮询。

## 2026-06-24 Codex/Provider UX 可达性修复

问题：Codex 状态检测同步等待 `codex doctor --json` 会让设置页长时间停在“检测中”，OpenAI/Anthropic Provider 下拉项也容易让用户误以为已接入余额/用量遥测。

根因：
- Codex doctor 是深度诊断，不适合作为设置页首屏的同步阻塞依赖。
- Provider 配置与 Pulse 遥测能力没有显式区分，保存 API Platform Key 容易被误读为 AI widgets 已启用。
- 主题选择器保留 HTML 静态默认项后又追加已安装主题，会出现重复默认主题和含糊的 Legacy 标签。

教训：
- Codex 状态接口默认只做快速 CLI/version/login 探针，doctor 结果必须走后台刷新和内存缓存。
- OpenAI/Anthropic 当前是 `saved_only`，只能标注为“配置已保存，遥测未接入”，不能暗示可查询 Codex/ChatGPT 订阅额度。
- 本地主题应标为“本地热更新”，旧主题应标为“旧主题兼容”，不得用泛化 Legacy 让用户猜含义。

排除清单：不得让 Settings 首屏被 Codex 深度诊断阻塞；不得把 OpenAI/Anthropic 保存成功写成遥测启用；不得让“打开 Codex 用量页”把 Pulse 当前 WebView 导航离开应用；不得出现重复主题选项。

## 2026-06-23 Runtime 验收修复

问题：Dashboard 新增 6 个主视觉 Canvas 没有初始化，硬件页普通 shim 图表在隐藏/切换场景中可能保持 1x1，首次设置 overlay 和 AI widget 自动隐藏会造成“功能消失”的错觉。

根因：UI 重构只更新了 DOM 和视觉结构，运行时 controller、URL tab、空态策略和隐藏 Canvas 性能策略没有同步落地。

教训：
- 新增 Canvas DOM 必须同时新增 controller、数据接入、resize/update 和 Playwright 像素采样验收。
- Tab 激活必须统一调度，不能只靠点击事件；URL 参数和 hash 也属于验收路径。
- 兼容 shim 不是 Chart.js，本地 shim 必须主动补齐 canvas 容器尺寸行为。
- 首次设置、AI 未配置、商店离线必须降级为可解释状态，不能用全局遮罩或删除组件来表达。

排除清单：不得再次让 Dashboard 主视觉 Canvas 空白；不得让硬件/分析图表停留在 1x1；不得把未配置 API key 解读为删除 AI widgets；不得让 8081 商店离线表现为空白市场。

## 2026-06-23 设计系统重置

问题：旧文档链持续把后续实现拉回构成主义和 Chart.js 思维。

根因：PRODUCT/DESIGN/AGENTS/交接文档/需求文档中同时存在旧约束和新目标，执行 agent 会优先套用旧禁令。

教训：设计系统发生根本重置时，必须同步权威文档，并给历史资料显式标注 legacy。

排除清单：不得继续使用旧构成主义作为当前 UI 目标；不得把 Chart.js 作为可见图表层默认方案；不得把自由卡片墙作为首页默认骨架。
