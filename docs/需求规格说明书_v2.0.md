## 一、版本定位

v1.0 完成了基础架构（Tauri + Python + WebSocket + 7-Tab SPA）。
v2.0 基于 Deepseek API 审查结论进行重大架构调整：

1. **Token 用量采集** — 从自动拉取变为手动 CSV 导入分析
2. **UX 重构** — 7 Tab → 5 Tab，仪表盘改为可配置组件墙
3. **插件架构** — Token 抓取改为可拓展插件，支持前端管理
4. **主题系统** — 模块化深度定制，预留主题市场商业化
5. **AI 供应商抽象** — 支持 Deepseek 外的其他 AI 供应商

## 三、各页面详细规格

### 3.1 仪表盘 — 可配置组件墙

**定位：** 开机默认展示，2m 等效视距可扫视。

**核心机制：**
- 组件卡片墙布局，用户可自由添加/移除/排列组件
- 每个组件有 **3 种尺寸** (S/M/L)，如 HarmonyOS 小组件
  - **S (1x1)** — 单数值 + 迷你趋势线（如 CPU 当前使用率 + 过去 60s 火花图）
  - **M (2x1 或 1x2)** — 较大图表或双数值并排（如内存饼图 + 数值）
  - **L (2x2)** — 完整独立图表（如 CPU 历史折线图）
- 拖拽排列，布局持久化到 localStorage
- **自适应 AI 区块：**
  - 如果 `config.json` 中 API Key 已配置 → 显示 AI 余额组件（实时 WebSocket 推送）
  - 如果 Key 为空 → 隐藏 AI 区块，将空间留给系统组件
- 几乎不出现纯文字表格，一切以动态图表为主

**预设可用组件：**

| 组件名 | 类型 | 数据来源 | 尺寸 |
|--------|------|----------|------|
| CPU 使用率 | 进度环 + 迷你火花图 | system WebSocket | S/M |
| 内存使用率 | 进度环 + 数值 | system WebSocket | S |
| 磁盘使用率 | 柱状迷你图（按分区） | system WebSocket | M |
| 网络速率 | 双轴迷你折线图（↑↓） | system WebSocket | M |
| GPU 温度 | 数值 + 色温指示 | system WebSocket | S |
| AI 余额 | 大号数值 + 货币符号 | deepseek WebSocket | S/M |
| 今日 Token 消耗 | 数值 + 消费进度条 vs 日限额 | SQLite 汇总 | M |
| 缓存命中率 | 环形图 | SQLite 汇总 | S |
| 系统正常运行时间 | 数字计时器 | system WebSocket | S |

**组件管理 UI：**
- 仪表盘右上角固定 "自定义" 按钮（齿轮图标）
- 点击进入编辑模式：
  - 每个组件出现删除 (×) 和尺寸切换 (↔) 按钮
  - 底部弹出可用组件库（未添加的组件以灰色态展示）
  - 拖拽组件交换位置
  - 点击 "完成" 保存布局

### 3.3 分析 — CSV 驱动数据分析

**定位：** 唯一的手动数据导入入口，纯离线分析页。

**页面布局：**
```
┌──────────────────────────────────────────────────┐
│  汇总卡片行                                       │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌──────────┐  │
│  │总Token │ │总费用   │ │缓存命中率│ │主要模型  │  │
│  │1.2M    │ │¥5.43   │ │78%     │ │v4-flash  │  │
│  └────────┘ └────────┘ └────────┘ └──────────┘  │
├──────────────────────────────────────────────────┤
│  筛选器: [日期范围选择器] [模型多选下拉]          │
│          [对比模式开关] [时间范围A] vs [时间范围B]│
├──────────────────────────────────────────────────┤
│  导入区 (仅在无数据或用户主动触发时显示)          │
│  ┌──────────────────────────────────────┐        │
│  │  📁 拖拽 .zip / .csv 文件到此处      │        │
│  │  或 [选择文件]                       │        │
│  │  支持 Deepseek Console 导出的 zip    │        │
│  └──────────────────────────────────────┘        │
├──────────────────────────────────────────────────┤
│  4 图表网格 (2x2)                                │
│  ┌──────────────────┐ ┌──────────────────┐      │
│  │ Token 趋势折线图  │ │ 模型分布柱状图   │      │
│  │ (对比模式双色)   │ │                 │      │
│  ├──────────────────┤ ├──────────────────┤      │
│  │ 缓存命中率环形图  │ │ 费用趋势折线图   │      │
│  └──────────────────┘ └──────────────────┘      │
├──────────────────────────────────────────────────┤
│  数据表格                                        │
│  ┌──────────────────────────────────────────────┐│
│  │日期│模型│输入Token│输出Token│缓存Token│总Token│费用│
│  │ ... │... │  ...   │  ...   │  ...   │ ...  │... │
│  └──────────────────────────────────────────────┘│
└──────────────────────────────────────────────────┘
```

**导入流程（智能识别 + 去重）：**
1. 用户拖入 `.zip` 或 `.csv` 文件
2. 系统检测文件类型：
   - `.zip` → 自动解压 → 扫描内所有 `.csv`
   - `.csv` → 直接解析
3. 智能适配列名（支持中英文、多种命名格式）
4. 自动去重：以 (日期 + 模型) 组合去重，已存在记录跳过
5. 导入完成 → 自动刷新汇总卡片 + 图表 + 表格
6. Toast 通知导入结果（成功 N 条 / 去重跳过 M 条 / 错误 0 条）

**图表规格：**
- 全部使用 Chart.js 4.x，红/黑/灰三色
- 硬边直角 (`borderRadius: 0`)，无渐变填充
- 对比模式下使用双色（红色 + 黑色）
- 支持时间范围缩放（7天 / 30天 / 自定义）

## 四、主题系统架构

### 设计目标
- 模块化加载 — 主题是独立 JSON/CSS 文件，可热加载
- 深度定制 — 颜色/字体/边框/阴影/图表配色/组件圆角全部可配
- 主题市场 — 预留主题打包/分发/购买机制

### 主题文件格式 (theme.json)
```json
{
  "name": "苏维埃构成主义",
  "version": "1.0.0",
  "author": "Pulse Team",
  "tokens": {
    "colorPrimary": "#CC0000",
    "colorBackground": "#000000",
    "colorSurface": "#1A1A1A",
    "colorText": "#FFFFFF",
    "colorTextSecondary": "#808080",
    "colorAccent": "#FFD700",
    "colorSuccess": "#2D8A2D",
    "fontDisplay": "'Tektur', 'Impact', sans-serif",
    "fontBody": "'Montserrat', 'Inter', sans-serif",
    "fontMono": "'JetBrains Mono', monospace",
    "borderRadius": "0px",
    "shadowStyle": "offset-only",
    "chartColors": ["#CC0000", "#000000", "#4D4D4D"]
```

### 主题优先级
1. 用户自定义覆盖 (localStorage)
2. 已安装主题 (theme.json)
3. 默认主题 (苏维埃构成主义)

## 五、数据流变更

### v1.0 数据流（废弃部分）
```
❌ collect_deepseek_loop() → GET /dashboard/usage → SQLite
```

### v2.0 数据流
```
✅ collect_deepseek_loop() → GET /user/balance → WebSocket → 仪表盘余额组件
✅ 用户手动下载 CSV zip → 拖入分析页 → 前端解析 → POST /api/csv/import → SQLite
✅ 分析页图表 ← SQLite deepseek_usage 表
✅ 插件(未来) → 统一 PluginBase 接口 → SQLite
```
