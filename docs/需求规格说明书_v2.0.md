## 一、版本定位

v1.0 完成了基础架构（Tauri + Python + WebSocket + 7-Tab SPA）。
v2.0 基于 Deepseek API 审查结论进行重大架构调整：

1. **Token 用量采集** — 从自动拉取变为手动 CSV 导入分析
2. **UX 重构** — 7 Tab → 5 Tab，仪表盘改为可配置组件墙
3. **插件架构** — Token 抓取改为可拓展插件，支持前端管理
4. **主题系统** — 模块化深度定制，预留主题市场商业化
5. **AI 供应商抽象** — 支持 Deepseek 外的其他 AI 供应商

## 三、各页面详细规格

### 3.1 仪表盘 — 可配置组件墙

**定位：** 开机默认展示，2m 等效视距可扫视。

**核心机制：**
- 组件卡片墙布局，用户可自由添加/移除/排列组件
- 每个组件有 **3 种尺寸** (S/M/L)，如 HarmonyOS 小组件
  - **S (1x1)** — 单数值 + 迷你趋势线（如 CPU 当前使用率 + 过去 60s 火花图）
  - **M (2x1 或 1x2)** — 较大图表或双数值并排（如内存饼图 + 数值）
  - **L (2x2)** — 完整独立图表（如 CPU 历史折线图）
- 拖拽排列，布局持久化到 localStorage
- **自适应 AI 区块：**
  - 如果 `config.json` 中 API Key 已配置 → 显示 AI 余额组件（实时 WebSocket 推送）
  - 如果 Key 为空 → 隐藏 AI 区块，将空间留给系统组件
- 几乎不出现纯文字表格，一切以动态图表为主

**预设可用组件：**

| 组件名 | 类型 | 数据来源 | 尺寸 |
|--------|------|----------|------|
| CPU 使用率 | 进度环 + 迷你火花图 | system WebSocket | S/M |
| 内存使用率 | 进度环 + 数值 | system WebSocket | S |
| 磁盘使用率 | 柱状迷你图（按分区） | system WebSocket | M |
| 网络速率 | 双轴迷你折线图（↑↓） | system WebSocket | M |
| GPU 温度 | 数值 + 色温指示 | system WebSocket | S |
| AI 余额 | 大号数值 + 货币符号 | deepseek WebSocket | S/M |
| 今日 Token 消耗 | 数值 + 消费进度条 vs 日限额 | SQLite 汇总 | M |
| 缓存命中率 | 环形图 | SQLite 汇总 | S |
| 系统正常运行时间 | 数字计时器 | system WebSocket | S |

**组件管理 UI：**
- 仪表盘右上角固定 "自定义" 按钮（齿轮图标）
- 点击进入编辑模式：
  - 每个组件出现删除 (×) 和尺寸切换 (↔) 按钮
  - 底部弹出可用组件库（未添加的组件以灰色态展示）
  - 拖拽组件交换位置
  - 点击 "完成" 保存布局


**导入流程（智能识别 + 去重）：**
1. 用户拖入 `.zip` 或 `.csv` 文件
2. 系统检测文件类型：
   - `.zip` → 自动解压 → 扫描内所有 `.csv`
   - `.csv` → 直接解析
3. 智能适配列名（支持中英文、多种命名格式）
4. 自动去重：以 (日期 + 模型) 组合去重，已存在记录跳过
5. 导入完成 → 自动刷新汇总卡片 + 图表 + 表格
6. Toast 通知导入结果（成功 N 条 / 去重跳过 M 条 / 错误 0 条）

**图表规格：**
- 全部使用 Chart.js 4.x，红/黑/灰三色
- 硬边直角 (`borderRadius: 0`)，无渐变填充
- 对比模式下使用双色（红色 + 黑色）
- 支持时间范围缩放（7天 / 30天 / 自定义）

## 四、核心插件：LAN 设备监控

### 4.1 设计理念

> **Pulse 就是 Agent。** 不需要额外安装任何软件。两台设备都安装 Pulse 并启用 LAN 插件后，即可在局域网内互相发现、配对、共享数据。

### 4.2 架构模型

```
┌─ 主控端 (Pulse Master) ───────────────┐
│  硬件页 ← 聚合展示                      │
│    ├── 本地系统 (完整)                  │
│    └── 横栏: 被控端A / 被控端B / ...    │
│  WebSocket Server ← 接收推送            │
│  配对管理器 ← UDP 广播发现              │
└────────────────────────────────────────┘
             ↕ WebSocket + UDP
┌─ 被控端A (Pulse Slave) ───────────────┐
│  WebSocket Client → 推送数据            │
│  指标选择器 → 用户授权哪些指标          │
│  系统托盘常驻 → GUI关闭后继续推送       │
└────────────────────────────────────────┘
```

- 同一设备可以同时是主控端（接收他人数据）和被控端（推送数据给他人）
- 角色由用户在插件页选择，非强制

### 4.3 配对流程

```
主控端                                被控端
  │                                     │
  │  1. 点击 [+] → 开始扫描             │
  │  ┌──────────────────────┐          │
  │  │ 正在扫描局域网...      │          │
  │  │ ● DESKTOP-ABC (在线)  │          │
  │  │ ● LAPTOP-XYZ (在线)   │          │
  │  └──────────────────────┘          │
  │                                     │
  │  2. 选择 "DESKTOP-ABC"             │
  │  ──── UDP 配对请求 ─────────────→  │
  │                                     │  3. 系统通知弹窗
  │                                     │  ┌──────────────────┐
  │                                     │  │ Pulse 监控请求    │
  │                                     │  │                   │
  │                                     │  │ "PULSE-SURFACE"  │
  │                                     │  │ 请求监控本设备    │
  │                                     │  │                   │
  │                                     │  │ ☑ 持久信任        │
  │                                     │  │ (需输入PIN验证)   │
  │                                     │  │                   │
  │                                     │  │ [允许]  [拒绝]    │
  │                                     │  └──────────────────┘
  │                                     │
  │  4. ←── 配对响应 (Token) ────────  │  用户点 [允许]
  │                                     │  生成配对 Token
  │  5. WebSocket 连接建立              │
  │  ←───── 指标数据流 ────────────    │  开始推送
```

**持久信任：**
- 用户勾选 ☑ 持久信任 → 输入 Windows 系统密码或 PIN → 系统验证
- 验证通过 → Token 持久化到被控端本地 → 后续自动授权
- 持久信任在以下情况失效：主控端软件卸载 / 被控端手动撤销 / 被控端系统重装
- 未勾选持久信任 → Token 有效期至主控端关闭或设备关机

**设备发现：**
- 主控端发送 UDP 广播报文到局域网 `255.255.255.255:PULSE_PORT`
- 所有启用 LAN 插件的 Pulse 实例监听同一端口
- 收到广播后回复：`{device_name, hostname, ip, plugin_version}`
- 主控端收集回复并展示可配对列表
- 扫描持续 5 秒后自动停止（可手动刷新）

### 4.6 开机自启与自动重连

#### 4.6.1 开机自启

**功能：** Windows 启动后 Pulse 自动运行并最小化到系统托盘。

**实现：** 利用已有的 `tauri-plugin-autostart`（已在 Cargo.toml + lib.rs 注册）。

**用户配置入口：** 配置页 → 系统卡片 → 开机自启开关
```
┌────────────────────────────────────────────┐
│  系统                                      │
│  ☑ 开机自动启动 Pulse                      │
│    启动后最小化到系统托盘                    │
│  ☑ 启动后自动连接已配对的主控端             │
│    (需要 LAN 插件已启用且有持久信任)         │
└────────────────────────────────────────────┘
```

**要求：**
- 默认关闭 — 用户主动开启
- 开启后 Pulse 注册到 Windows 启动项
- 启动时带 `--minimized` 参数，直接进入托盘模式

#### 4.6.2 自动重连主控端

**功能：** Pulse 启动后，如果 LAN 插件已启用且有持久信任的配对记录，自动尝试连接主控端，无需用户手动操作。

**流程：**
```
Pulse 启动
  ↓
LAN 插件初始化
  ↓
读取本地配对记录 (localStorage / config)
  ├── 无持久信任记录 → 跳过，不做任何事
  └── 有持久信任记录 → 开始重连
       ↓
     对每条记录:
       ├── 发送 UDP ping 到主控端 IP
       │    ├── 主控端在线 → WebSocket 连接 → 恢复推送
       │    └── 主控端离线 → 每 30s 静默重试 (最多重试 10 次)
       └── 10 次全部失败 → 放弃本次启动周期
           (下次 Pulse 重启时重新尝试)
```

**重连策略：**
- UDP ping 超时: 3 秒
- 重试间隔: 30 秒
- 最大重试次数: 10 次 (共 5 分钟)
- 超过最大次数 → 放弃，不在前台显示任何错误
- 被控端托盘图标显示连接状态 (绿点/黄点/灰点)
- 主控端重新上线后 → 被控端在下一个重试周期自动连上

**连接状态指示 (托盘/UI)：**
| 状态 | 图标 | 说明 |
|------|------|------|
| 已连接 | 🟢 | 成功连接主控端，正在推送 |
| 重连中 | 🟡 | 主控端离线，正在重试 |
| 未配对/离线 | ⚪ | 无持久信任记录或已放弃重连 |

**配置选项 (插件页 → LAN 插件卡片)：**
```
┌──────────────────────────────────────────────┐
│ 🖥️ LAN 设备监控                      ● 已启用 │
│ ...                                          │
│ 启动行为:                                    │
│   ☑ 启动后自动连接主控端                     │
│   ☐ 连接失败时显示通知                       │
│   [配置重连策略...]                           │
│                                              │
│   ┌─ 高级重连设置 ─────────────────────┐     │
│   │  重试间隔: [30] 秒                  │     │
│   │  最大重试: [10] 次                  │     │
│   │  ☐ 后台静默重连 (推荐)              │     │
│   │  ☐ 前台显示重连进度                 │     │
│   └────────────────────────────────────┘     │
└──────────────────────────────────────────────┘
```

**安全约束：**
- 只有持久信任 (PIN/密码验证过的) 配对记录才自动重连
- 临时配对 (未勾选持久信任) 的 Token 在 Pulse 退出时即失效，不会自动重连
- 用户可随时在插件页手动断开 → 清除持久信任 → 下次不再自动重连
