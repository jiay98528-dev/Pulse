## 一、项目概况

Surface Go Win10 上的 AI Telemetry / Operations Surface，同时监控本机系统状态、Deepseek/AI Provider 用量和 LAN 设备健康度。

当前前端方向为次世代动态监控软件：自研 Canvas telemetry 图表、半固定可配置首页、主题 schema v3。旧苏联构成主义视觉风格为 legacy，不再作为当前约束。

## 三、技术架构决策

### 最终选型（2026-06-19 确定）

```
Tauri v2 (Rust + WebView2)
  ├── WebView ← frontend/ (HTML/CSS/JS + TelemetryCanvas)
  ├── Tauri Plugins ← 系统托盘/自启/全局热键
  └── Sidecar ← Python backend/main.py (数据采集层)
       ├── psutil → 本机系统 (CPU/内存/磁盘/网络)
       ├── WMI → GPU/温度/远程设备
       └── aiohttp → Deepseek API
```

### 决策理由（已确认）

1. **Tauri > Electron**: Surface Go 4GB RAM，Tauri 省 100MB+。Electron 的 Chromium 单独占用 150-200MB，WebView2 是系统组件仅 20-40MB
2. **Tauri > PyQt6**: 前端 HTML/CSS/JS 代码零改动，自研 TelemetryCanvas 可控、低内存，适配 Surface Go 长时间运行
3. **Python 后端不变**: psutil / WMI / Deepseek SDK 只有 Python 版，Sidecar 模式是最小侵入方案
4. **纯 Web > 桌面套壳**: 浏览器打开 `localhost:8080` 仍然可用，Tauri 只是增强体验（托盘/自启/全屏）

### 数据流

```
Python (每秒) → WebSocket → TelemetryCanvas 实时更新
Python (每30秒) → SQLite 持久化 → Deepseek 历史查询
Frontend (用户操作) → HTTP API → Python (配置保存/CSV导入)
```

### 正式发行边界

- 本地开发运行：`venv/Scripts/python.exe backend/main.py`，前端由源码 `frontend/` 提供。
- 可构建安装包：`scripts/build-release.ps1` 生成 PyInstaller backend sidecar，再执行 `cargo tauri build`。
- 可正式发行：还必须完成安装后 smoke，即从 MSI/NSIS 安装目录启动 Pulse，确认无源码、无系统 Python、无仓库 venv 时后端自动启动。
- 运行时环境变量：
  - `PULSE_DATA_DIR`：发行态配置、SQLite 和运行态文件写入目录。
  - `PULSE_FRONTEND_DIR`：覆盖前端静态资源目录；frozen 默认使用 `_MEIPASS/frontend`。
  - `PULSE_HOST` / `PULSE_PORT`：sidecar HTTP 监听覆盖。
- 主题商店 8081 是可选外部服务；主应用必须显示离线态，本次主安装包不内置商店服务。

## 四、核心边界与约束

### 硬件约束
- Surface Go 第一代: Intel Pentium Gold, 4GB RAM, 10" 1280×800
- 等效视距: 物理 40cm ≈ 27" 显示器 2m → **大字号高对比**
- 触屏操作: 所有交互热区 ≥ 44px

### 设计约束（来自错题本）
1. 当前方向是 AI Telemetry / Operations Surface，旧构成主义只作 legacy 历史参考。
2. 动效必须服务数据刷新、趋势、阈值、在线/离线或用户操作反馈。
3. 远距扫视可读：正文最小 14px，关键 KPI 48px 以上，触屏热区不小于 44px。
4. Canvas 历史点有上限，隐藏 tab 不参与 RAF，不允许 1x1 Canvas 成为最终布局。
5. 未配置 API key、商店离线等状态必须显示解释性空态，不能静默隐藏功能。

### 技术约束
- Python 3.10+，不使用 GPU/NVIDIA 专有库
- 前端无框架依赖（纯原生 JS + 本地 TelemetryCanvas）
- WebSocket 与 HTTP 同源使用 8080，Tauri/非本地页面通过 `http://127.0.0.1:8080` fallback
- Deepseek API: 官方 `https://api.deepseek.com`，Key 存在 `config.json`
- ⚠️ Deepseek 无公开用量监控 API — `/dashboard/usage` 是非公开接口，需改为本地拦截模式
- 主力机数据通过 WMI 远程获取（后续配置）

## 五、关键流程片段

### 启动流程
```python
# backend/main.py — 当前入口
startup:
  1. init_db()          # SQLite 建表
  2. create_task(system_loop)     # 每秒系统数据 → WebSocket
  3. create_task(deepseek_loop)   # 每30秒 Deepseek → WebSocket + SQLite

# Tauri release 流程:
Tauri 启动 →
  1. debug: 拉起 src-tauri/scripts/start-backend.bat
  2. release: 拉起 bundled sidecar pulse-backend
  3. sidecar 写入 PULSE_DATA_DIR，不写安装目录
  4. WebView 加载 frontendDist，JS 连接 ws://127.0.0.1:8080/ws
```

### 数据采集流程
```
collect_system_loop()
  ├── psutil: CPU/内存/磁盘/网络/电池
  ├── WMI: GPU/温度
  ├── 计算网速差值
  └── broadcast({"type": "system", "data": ...})

collect_deepseek_loop()
  ├── GET /user/balance → 余额 (公开API ✅)
  ├── GET /dashboard/usage → CSV zip 下载 (⚠️ 非公开内部接口，API Key 鉴权)
  ├── 解压 zip → 2个CSV: 用量明细.csv + 费用明细.csv
  ├── 解析 CSV → SQLite 持久化
  ├── SQLite: 汇总今日/本周/本月/分模型
  └── broadcast({"type": "deepseek", "data": {...}})

⚠️ 架构约束 (2026-06-20 最终决策):
  Deepseek 公开 API（4个端点）中没有 Token 用量聚合查询。当前采用路线：
  定时(1-2h)调 /dashboard/usage → 获取 zip → 解压解析 → SQLite。
  若未来被封/下线: 降级为纯余额监控 + 手动 CSV 导入。
  合规: 使用本人 API Key 鉴权，非攻击行为。
  详见: docs/CSV_ZIP自动导入方案.md
```

### CSV 导入流程
```
用户拖拽/选择 CSV → 前端解析列名映射 →
POST /api/csv/import (multipart) →
Pandas 读 CSV → 字段匹配（7个标准列） →
批量 INSERT → 返回导入数量 → 前端刷新历史图表
```

### WebSocket 消息格式
```json
// 系统数据（每秒）
{"type": "system", "data": {"cpu": {...}, "memory": {...}, "disk": [...], "gpu": [...], "network_speed": {...}, ...}}

// Deepseek 数据（每30秒）
{"type": "deepseek", "data": {"balance": {...}, "today": {...}, "week": {...}, "month": {...}, "model_breakdown": [...], "history_7d": [...], "limits": {...}, "over_limit_daily": false}}
```

### 前端页面结构
```
Tab 仪表盘:
  ├── Telemetry Hero: AI 余额/系统负载/数据新鲜度
  ├── Core Panels: 系统流、AI 用量流、热度条
  └── Widget Dock: CPU/内存/磁盘/网络/GPU/AI widgets，可增删排序

Tab 硬件:
  ├── CPU 每核负载、内存、磁盘、网络、GPU/温度、电池图表
  ├── 状态横幅
  └── LAN 设备横栏 + 抽屉详情

Tab 分析:
  ├── CSV/ZIP 导入
  ├── 汇总 KPI、趋势图、模型分布、缓存率、费用趋势
  └── 历史表格与筛选器

Tab 配置:
  ├── AI Provider/API Key/限额配置
  ├── 主题 schema v3 编辑器与市场入口
  └── 主题商店 8081 离线态

Tab 插件:
  ├── 插件启停
  └── LAN 配对与共享指标配置
```
