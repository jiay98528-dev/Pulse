# Product

## Register

product

## Users

一位在 Surface Go 第一代（Win10，10英寸屏）上使用本看板的开发者。物理视距约 40cm，但等效于在 27 寸显示器上 2m 处观看，因此文字和视觉元素必须放大、对比度必须极高。

主要使用场景：开发时在副屏位置（Surface Go 旁置）持续显示，远距扫视关键指标，偶尔靠近点击切换视图。每晚/每周查看趋势汇总。

未来将扩展至同一局域网下的主力开发机系统监控。

## Product Purpose

在 Surface Go 上建立一个兼具工具性和视觉冲击力的实时数据看板：
- 实时监控本机（以及未来的局域网设备）系统状态——CPU、内存、GPU、温度、磁盘、功率
- 实时展示 Deepseek API 用量——Token、缓存命中率、余额、资费
- 每秒同步数据，WebSocket 实时推送
- 支持 CSV 导入补充历史趋势数据
- 开机自启，作为常驻辅助屏幕

成功标准：每天至少看一眼就获取到需要的系统状态和 API 用量信息，不产生额外操作负担。

## Brand Personality

**热血 · 力量 · 秩序**（Passionate, Powerful, Orderly）

- **热血**：苏联构成主义的革命能量，红色楔形穿刺白色圆形的进攻性，罗德琴科式的对角线动态，数据流如红旗飘扬般有生命感
- **力量**：粗重字体、厚实边框、硬边几何、大号数值——每一个视觉元素都有重量感，不让任何设计显得纤弱或轻浮
- **秩序**：虽然是热血风格，但数据层级清晰、信息架构有序。革命不是混乱，是旧秩序的打破和新秩序的建立

## Anti-references

- **不简陋**：不要像终端面板或 minimal 命令行工具。构成主义的特点是精心构造的视觉密度——每个几何块都是有意安排的，不是「省事」的产物
- **不浮夸**：不要渐变、发光、毛玻璃、圆滑边框。构成主义的粗粝感来自硬边直角的坦诚，不是过度装饰
- **不赛博**：不要纯黑+霓虹绿/蓝的「黑客帝国」或「赛博朋克」风格。这是1920年代的工厂美学，不是2049年的夜店美学

## Design Principles

1. **动态活力优先**。实时刷新是核心特征——数据流动感、图表动画、状态脉冲是「热血」个性的主要载体。静态看板就是失败的看板。

2. **远距可读 = 大胆排版**。Surface Go 的小屏等效远距观看，逼迫设计必须使用大字号、高对比、极简色彩层级。如果某个元素在屏幕这头看不清，它就不该存在。

3. **信息密度有节奏**。尽管是高密度数据看板，但每个区块的视觉重量必须有差异——主力指标占大块，次要指标缩小，制造画面内的「景深感」。

4. **诚实构造**。每个视觉元素都有功能目的。装饰性几何（斜线、红块、五角星）同时承担视觉引导/数据分隔/状态指示的功能，不产生无意义的视觉噪声。

5. **离线韧性**。Deepseek API 偶尔不可用时，看板仍然展示本地系统数据和缓存的历史指标，不崩溃不白屏。数据显示降级而非失败。

## Accessibility & Inclusion

- Surface Go 10 寸屏 + 等效 2m 视距 → 最小字号不低于 14px（正文字号建议 16px+）
- 极高高对比度：纯白文字（#FFFFFF）在极深黑背景（#0A0A0A）上
- 红/黑/白三色主视觉：红色不单独作为信息传递的唯一方式（配合文字标签）
- 色彩盲友好：主要数据区分靠位置/标签而非仅靠颜色
- 无闪烁动画避免光敏风险，实时更新采用平滑过渡而非突变
